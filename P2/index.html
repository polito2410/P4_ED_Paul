Hasta el origen y mas alla 7

Hay un gran abuso del parametro "origin"(El nombre que recibe el repositorio remoto principal) ya que git puede especificar a la nada como un origen el cual lo da como valido git a "git push"(repositorio local a repositorio remoto) y tambien a "git fetch" (Descargar contenidos desde repositorio remoto)
Para especificar la nada se usa:"git push origin :"x" " o "git fetch origin :"x" " en las 2 hay que poner un parametro vacio para que se especifique la nada.


En esta imagen vemos un origen a "nada" de una rama remota. Para eliminar esta se usa "git push origin :foo" (sin olvidar el parametro vacio)

En esta imagen vemos que la rama "foo" se ha eliminado , usando lo dicho anteriormente: "git push origin :foo"

Se hace fetch a "nada" en un lugar local , crea otra rama. Para hacer esto se usa "git fetch origin :bar"(sin olvidar otra vez el parametro vacio)
